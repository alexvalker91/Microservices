Коротко о Docker:
 1) Докер контейнер:
   1.1) Это легковесный, исполняемый пакет программного обеспечения, который включает все необходимое для запуска приложения: код, среду выполнения, системные инструменты, системные библиотеки и настройки.
   1.2) Контейнеры изолируют приложение от окружающей инфраструктуры, что обеспечивает единообразие работы в разных средах (напр., на вашем компьютере, на сервере, в облаке).
 2) Докер образ (Image):
   2.1) Шаблон для создания контейнеров. Образ читается только и содержит пошаговые инструкции для создания контейнера. Обычно образы хранятся в реестрах (например, Docker Hub).
 3) Dockerfile:
   3.1) Текстовый файл с инструкциями для сборки образа. В нем указывается базовый образ, команды для установки зависимостей, копирования файлов приложения и т.д.
 4) Docker Compose:
   4.1) Инструмент для определения и запуска многоконтейнерных приложений Docker. Вы описываете сервисы, сети, тома в файле docker-compose.yml, и затем одной командой запускаете всё вместе.

Как у меня это работает:
- Dockerfile для каждого микросервиса → описывает как упаковать приложение в образ
- docker-compose.yml → описывает как запустить все образы вместе
- Контейнеры → запущенные экземпляры образов
- Сети → виртуальные "провода" между контейнерами
